Serial Mode Communication

  • Components
  • March 11, 2025

UART Points

  • UART uses two wires - Tx & Rx
  • Communication can be simplex, duplex or full-duplex
  • Data is tranmsitted in frames.
  • UART is simple, low cost and easy to implement.
  • There is no common clock signal. Since there is no clock, the devices must transmit and receive at the same known speed which is nothing but baudrate.
  • UART doesn’t define what level of voltage is considered HIGH and below what voltage the signal is considered LOW.

Arduino

  1. Run the following code in Arduino:
void setup() {
  /* Initialise Serial Communication */
  Serial.begin(9600);
}

void loop() {
  /* Print a string every one second */
   Serial.print("Serial Data");
   delay(1000);
}

The code just prints a data every one second.

  1. In order to receive data from Arudino via Python run the following code:

Make sure you change the port according to your connection. In order to know which port the Arduino is using, open Device Manager in Windows:

import serial
import time

arduino = serial.Serial(port='COM4', baudrate=9600, timeout=.1)

while True:
    value   = arduino.readline()
    time.sleep(1)
    t = time.localtime()
    current_time = time.strftime("%H:%M:%S", t)
    print(current_time , end=" ") # Parameter `end` is used to print value in same line
    print(value)

USB to TTL

We will use a USB to TTL adapter with an Arduino UNO to understand what the adapter does. Yes, the Arduino UNO already has a built-in USB Interface chip, but for this exercise, we will assume that it is only used for uploading programs and not for reading serial data back.

TTL stands for Transistor Transistor Logic which maily operates at 3V/5V. You can see in the image below, the adapter allows both 3V and 5V to be set using jumper. We select 3V if the MCU we have operates at 3V VCC and 5V if it operates at 5V VCC. Since we are using an Arduino UNO, which operates at 5V supplied by the USB cable, we will set the jumper to 5V.

Next, we flash a simple Arduino program that prints serial data every second or so at 9600 baud. After uploading the program to the Arduino, we move on to wiring the connections as shown below:

Connection Summary:

  • Connect the USB-to-TTL adapter to the Arduino as shown in the diagram.
  • Plug the USB-to-TTL adapter into the computer.
  • Run the provided Python code with the correct COM port. You will be able to see the data on the python terminal.

Note

If the USB-to-TTL adapter is powered on while uploading a new program to the Arduino, it may interfere with the upload process by holding the TX and RX lines, causing the flashing to fail. To avoid conflicts during flashing, disconnect the USB-to-TTL adapter while uploading new programs.

Use Arduino as USB to TTL

The next step would be to use Arduino as the USB to TTL converter. Use another MCU as the serial signal generator and Arduino as the converter and receive data to the computer.

STM32

I was given this new USB to RS485 and UART converter by https://github.com/even-notodd/ and I wanted to test it out with my STM32 Nucelo board.

The converter is an open hardware design and can be accessed here: https://github.com/even-notodd/ISO-RS485

  1. Open STM32 Project and enable any USART/UART. I used UART with PF6 and PF7.
  2. Connected PF6 to TX of the converter and PF7 to RX Pin of the converter. Connected the GND Pin of the connector to one of the GND pin on the STM32 Nucelo board.
  3. Connected to the USB-C port on the board to PC.
  4. Modified code to send data over UART7 and verified the receiving of messages in TeraTerm.

Note to Self: The jumper on the board is not for UART.

Queries

Why do you need a USB to TTL ?

  • To establish a UART connection between a microcontroller board and a PC when the board lacks a built-in USB-to-Serial converter.
  • It enables programming and serial communication with custom boards without needing extra permanent hardware.

What is the difference between RS232 & TTL ?

  • Voltage levels: TTL operates at 0–3.3V/5V, while RS232 operates at ±3V to ±15V.
  • Signal inversion: TTL is non-inverted (HIGH = 1, LOW = 0), whereas RS232 signals are inverted.
  • Application: TTL is used inside microcontrollers and digital circuits, while RS232 is meant for longer distance communication (e.g., between PCs and peripherals).

What is the difference between RS485 and UART ?

  • UART = the data format and timing (a protocol)
  • RS-485 = the electrical signaling standard (hardware layer)
